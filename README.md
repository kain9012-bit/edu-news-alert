# 교육동향 보도자료 알리미

교육부와 시도교육청 보도자료를 매일 수집해, 직전 24시간의 교육동향을 Chrome·Edge 사이드바에서 확인하는 프로그램입니다. 이전 자료는 별도의 기간 조회 화면에서 검색할 수 있습니다.

## 현재 동작

- 매일 한국시간 오전 8시에 GitHub Actions가 수집을 시작합니다.
- 오전 8시를 기준으로 직전 24시간에 작성된 보도자료를 수집합니다.
- 같은 보도자료는 다시 저장하지 않습니다.
- 전체 자료는 최근 14일 동안 `public/news.json`에 보관합니다.
- 최신 24시간 브리핑은 `public/latest.json`에 별도로 만듭니다.
- 확장 프로그램은 자동 알림을 띄우지 않습니다. 사용자가 사이드바를 열면 최신 브리핑에서 관심 키워드와 일치하는 자료만 보여줍니다.
- 이전 자료는 시작일과 종료일을 직접 지정하거나 3일·7일·14일 단위로 조회할 수 있습니다.

GitHub Actions의 예약 실행은 GitHub 사정에 따라 오전 8시보다 몇 분 늦게 시작될 수 있습니다. 실행이 늦어져도 브리핑의 기준 시각은 오전 8시로 고정됩니다.

## 수집 대상

`crawler/sources.json`에는 21개 게시판이 등록되어 있으며, 확장 프로그램에서는 같은 기관의 게시판을 하나로 묶어 17개 기관으로 표시합니다.

- 교육부
- 전북특별자치도교육청
- 서울특별시교육청
- 경기도교육청
- 부산광역시교육청
- 대구광역시교육청
- 인천광역시교육청
- 전남광주통합특별시교육청
- 대전광역시교육청
- 울산광역시교육청
- 세종특별자치시교육청
- 강원특별자치도교육청
- 충청북도교육청
- 충청남도교육청
- 경상북도교육청
- 경상남도교육청
- 제주특별자치도교육청

전북특별자치도교육청과 전남광주통합특별시교육청은 여러 게시판을 수집하지만 화면에서는 각각 한 기관으로 표시됩니다. 인천광역시교육청의 묶음 보도자료는 개별 보도자료로 나누어 저장합니다.

## 확장 프로그램 사용

1. Chrome에서는 `chrome://extensions`, Edge에서는 `edge://extensions`를 엽니다.
2. 개발자 모드를 켭니다.
3. `extension` 폴더를 압축해제된 확장 프로그램으로 불러옵니다.
4. 확장 프로그램 아이콘을 누르거나 `Ctrl+Shift+Y`를 눌러 사이드바를 엽니다.

단축키는 Chrome의 `chrome://extensions/shortcuts` 또는 Edge의 `edge://extensions/shortcuts`에서 바꿀 수 있습니다.

사이드바에는 오전 8시 기준 직전 24시간 자료 중 선택한 기관과 관심 키워드 조건에 맞는 보도자료만 표시됩니다. 사이드바 검색창에 단어를 입력하면 저장된 관심 키워드와 관계없이 최근 24시간 전체 자료에서 해당 단어를 바로 찾고, 검색창을 비우면 관심 키워드 목록으로 돌아갑니다. 기사 제목을 누르면 새 탭에서 원문이 열리고 사이드바는 유지됩니다. `이전 보도자료 기간 조회`를 누르면 최근 14일 전체 자료를 날짜·기관·검색어·관심 키워드로 조회할 수 있습니다.

설정 화면에서는 다음 항목을 바꿀 수 있습니다.

- 표시할 기관
- 관심 키워드
- 키워드를 찾을 범위: 제목만 또는 제목과 내용 일부
- GitHub Pages 데이터 주소

관심 키워드는 사이드바에 표시할 보도자료를 고르는 조건입니다. 기간 조회 화면에서는 전체 자료를 확인하거나 특정 관심 키워드로 따로 거를 수 있습니다.

확장 프로그램 파일을 수정한 뒤에는 확장 프로그램 관리 화면에서 다시 로드해야 변경 내용이 적용됩니다.

## 데이터 파일

- `public/news.json`: 최근 14일 보도자료 전체
- `public/latest.json`: 오전 8시 기준 최신 24시간 브리핑과 조회 구간
- `public/sources.json`: 화면에 표시할 수집 기관
- `public/status.json`: 마지막 수집 시각, 보관 기간, 기관별 성공·실패 상태

이 프로젝트는 별도의 데이터베이스 서비스 대신 GitHub 저장소의 JSON 파일을 사용합니다. 오래된 자료는 매일 수집할 때 자동으로 제거됩니다.

게시기관이 작성 시각까지 제공하면 정확한 24시간 범위를 적용합니다. 날짜만 제공하는 게시판은 누락을 막기 위해 브리핑 시작일과 종료일에 해당하는 자료를 포함할 수 있습니다.

## 자동 수집과 배포

자동 수집은 `.github/workflows/collect-news.yml`에서 실행합니다.

- 예약 실행: 매일 `23:00 UTC`, 한국시간 다음 날 오전 8시
- 수동 실행: GitHub 저장소의 `Actions` → `Collect education press releases` → `Run workflow`
- 보관 기간: 14일
- 수집 범위: 오전 8시 기준 직전 24시간
- 배포: `public` 폴더를 GitHub Pages로 배포

워크플로가 수집 결과를 저장하려면 저장소의 Actions 설정에서 `Workflow permissions`가 `Read and write permissions`로 되어 있어야 합니다.

## 로컬 실행

Python 3.11 이상에서 다음 명령으로 수집기를 실행할 수 있습니다.

```powershell
python -m pip install -r requirements.txt
python crawler/collect.py
```

환경변수로 보관 기간과 수집 범위 등을 바꿀 수 있습니다.

- `RETENTION_DAYS`: 보관 일수, 기본값 14
- `COLLECTION_WINDOW_HOURS`: 브리핑 수집 범위, 기본값 24
- `BRIEFING_HOUR_KST`: 한국시간 브리핑 기준 시각, 기본값 8
- `MAX_ITEMS_TOTAL`: 전체 저장 건수 상한, 기본값 3000
- `MAX_ITEMS_PER_SOURCE`: 기관별 한 번에 확인할 링크 수, 기본값 20

## 현재 범위와 다음 단계

현재 버전은 보도자료 수집, 제목·내용 일부 추출, 최신 24시간 브리핑, 14일 기간 조회까지 구현되어 있습니다. 로컬 LLM을 이용한 보도자료 유형 분류와 교육동향 요약은 아직 연결하지 않았습니다. 이후 LLM 기능을 추가할 때도 원문 수집 결과와 모델이 만든 분류·요약을 구분해 저장할 예정입니다.

프로그램의 동작, 설정, 수집 대상 또는 보관 정책이 바뀌면 이 README도 같은 변경에서 함께 갱신합니다.
